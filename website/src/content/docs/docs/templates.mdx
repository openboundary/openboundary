## What's in a Template

Each template includes:

| Component | Description |
|-----------|-------------|
| `spec.yaml` | Complete OpenBoundary specification |
| `openapi.yaml` | Full API contract with request/response schemas |
| `src/db/schema.ts` | Drizzle database schema with all tables |
| `src/auth/model.conf` | Casbin authorization model |
| `src/auth/policy.csv` | Role-based access policies |
| `README.md` | Setup instructions and customization guide |

### Example: CRM Template Structure

```
crm-template/
├── spec.yaml              # 15+ components defined
├── openapi.yaml           # 40+ endpoints
├── src/
│   ├── auth/
│   │   ├── auth.config.ts # Session-based auth
│   │   ├── model.conf     # RBAC with teams
│   │   └── policy.csv     # Sales, manager, admin roles
│   └── db/
│       └── schema.ts      # contacts, companies, deals, activities
└── README.md
```

## Request a Template

Have a use case we haven't covered? [Open an issue](https://github.com/openboundary/openboundary/issues/new?template=template-request.md) with:

1. Application type and domain
2. Key entities and relationships
3. Authentication/authorization requirements
4. Compliance requirements (if any)

## Contributing Templates

Templates are community-driven. To contribute:

1. Fork the repository
2. Create a new template in `templates/`
3. Include all required files (spec, openapi, schema, auth)
4. Add documentation
5. Submit a pull request

See [CONTRIBUTING.md](https://github.com/openboundary/openboundary/blob/main/CONTRIBUTING.md) for guidelines.
