Quick Start

1

Install

Clone and build from source

git clone https://github.com/openboundary/openboundary.git && cd openboundary && go build -o openboundary ./cmd/openboundary
2

Create spec

Initialize a project

openboundary init basic -o my-project
3

Generate

Compile to TypeScript

openboundary compile spec.yaml -o generated
4

Run

Start the server

cd generated && npm install && npm run dev

Configuration

Property Type Description
frameworkrequired "hono" Web framework
portrequired integer Port number (1-65535)
openapi string Path to OpenAPI spec
middleware array Middleware chain
depends_on array Dependencies for injection

Example

spec.yaml 
- id: http.server.api
  kind: http.server
  spec:
    framework: hono
    port: 3000
    openapi: ./openapi.yaml
    middleware:
      - middleware.authn
    depends_on:
      - postgres.primary

  kind: middleware
  spec:
    provider: casbin
    depends_on:
      - middleware.authn
    model: ./src/auth/model.conf
    policy: ./src/auth/policy.csv

Use Case

usecase

Business requirements bound to routes with acceptance criteria.

Configuration

Property Type Description
binds_torequired string Route: server:METHOD:/path
goalrequired string What this accomplishes
middleware array Override middleware chain
actor string Who performs this
preconditions array Required conditions
acceptance_criteria array Success criteria
postconditions array State after execution

Example

spec.yaml 
- id: usecase.create-user
  kind: usecase
  spec:
    binds_to: http.server.api:POST:/users
    middleware: []  # Public endpoint
    goal: Register a new user
    actor: anonymous
    preconditions:
      - Email not already registered
    acceptance_criteria:
      - User created with email
      - Password hashed

Middleware override: Omit = full chain applies. [] = public. [middleware.authn] = specific subset.

CLI Reference

openboundary compile 
# Compile spec to code
openboundary compile <spec-file> [options]

  -o, --output <dir>   Output directory (default: ./generated)
  --dry-run            Show what would be generated
  --force              Overwrite existing files
openboundary validate 
# Validate without generating
openboundary validate <spec-file> [options]

  --strict    Warnings become errors
  --json      Output as JSON
openboundary init 
# Create from template
openboundary init [template] [options]

  Templates: basic, crud, multiservice
  -o, --output <dir>   Output directory
