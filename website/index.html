The Problem

AI writes code. Who guards the architecture?

AI assistants generate business logic fast. But they don't understand your security requirements, middleware chains, or dependency boundaries. You need invariants that can't be accidentally bypassed.

Enforce Security Invariants

Authentication and authorization requirements are declared once. Generated code enforces them. AI can't accidentally create an unprotected endpoint.

Separate Boundaries from Logic

Infrastructure boundaries (auth, routing, middleware) rarely change. Domain logic changes constantly. Keep them separate. Let AI focus on the high-entropy work.

Reduce Cognitive Load

Stop reviewing boilerplate. Stop checking if middleware is wired correctly. The spec is the source of truth. Regenerate when requirements change.

How it works

Specification to code

Declare your architecture. Generate the scaffolding. Write only the domain logic.

spec.yaml you write this 
version: "0.1.0"
name: user-api

components:
  - id: http.server.api
    kind: http.server
    spec:
      framework: hono
      port: 3000
      middleware:
        - middleware.authn # Always enforced
        - middleware.authz

  - id: usecase.create-user
    kind: usecase
    spec:
      binds_to: http.server.api:POST:/users
      middleware: [] # Explicit public
      goal: Register a new user
generated/src/server.ts generated 
// Generated by OpenBoundary - DO NOT EDIT
import { Hono } from 'hono'
import { authn } from './middleware/authn'
import { authz } from './middleware/authz'
import { createUser } from './handlers/users'

const app = new Hono()

// Protected routes (server-level middleware)
app.use('*', authn, authz)

// Public: createUser (explicit override)
app.post('/users', createUser)

export default app
