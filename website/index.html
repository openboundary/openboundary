How it works

Specification to code

Write a YAML spec. Get a complete, type-safe backend.

spec.yaml input 
version: "0.1.0"
name: user-api

components:
  - id: http.server.api
    kind: http.server
    spec:
      framework: hono
      port: 3000
      middleware:
        - middleware.authn

  - id: usecase.create-user
    kind: usecase
    spec:
      binds_to: http.server.api:POST:/users
      goal: Register a new user
generated/src/server.ts output 
// Generated by OpenBoundary
import { Hono } from 'hono'
import { authn } from './middleware/authn'
import { createUser } from './handlers/users'

const app = new Hono()

app.use('*', authn)
app.post('/users', createUser)

export default app
Features

Why OpenBoundary

Type-safe by default

Generated code is fully typed. OpenAPI schemas become TypeScript types.

Minutes, not hours

Skip the boilerplate. Declare your architecture once, generate everywhere.

Spec = documentation

Your specification is always accurate because it generates the code.

Real frameworks

Uses Hono, Drizzle, better-auth. No abstractions, no lock-in.

Validation built-in

Schema validation, cross-reference checking, circular dependency detection.

Open source

AGPL-3.0 licensed. Read the code, contribute features, fork for your needs.
